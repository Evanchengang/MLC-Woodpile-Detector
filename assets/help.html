MLC Woodpile Detector Pro - User Guide

1. Interface Overview

The application window is divided into four main areas:

Header (Top White Bar)

Displays the Marine Lumber Co. logo on the left, the system title "Industrial Wood Pile Detection System" in the center, and the version number below it. This area is for branding only and contains no interactive controls.

Configuration Panel (Left Side)

This is where you set up the detection job before running it. All input fields, buttons, and parameters are located here.

Monitoring Panel (Right Side)

Shows real-time status, progress, statistics, and a detailed processing log during detection. Use this area to track the current job.

Action Bar (Bottom)

Contains three large buttons: START DETECTION, OPEN OUTPUT, and HELP GUIDE.

2. Configuration Panel - Detailed Explanation

YOLO Model File (.pt)
Shows the path of the currently loaded YOLO detection model. On startup, the app automatically loads assets/best.pt.
Import button: Click to browse and select a different .pt model file from your computer. After selection, the app loads the new model and updates the status to "Model Ready".
Input Images Folder
The folder containing the original satellite or aerial images you want to analyze. Supported formats: .png, .jpg, .jpeg.
Browse button: Opens a folder picker dialog. Select the folder that contains your source images. The app will scan this folder and ignore any files that already start with detected_ (to avoid re-processing results).
Output Results Folder
The folder where annotated result images will be saved. Each output image is a copy of the original with green bounding boxes and confidence scores drawn on it.
Browse button: Opens a folder picker dialog. It is recommended to create a new empty folder for each batch job to keep results organized.
Confidence Threshold (0.0 - 1.0)
A slider that filters detection results by confidence score. Only detections with a confidence score equal to or higher than this value will be kept.
  • 0.25 - 0.30: Recommended for most clear satellite images. Balances precision and recall.
  • 0.10 - 0.15: Use when targets are small, blurry, or far away. Will catch more objects but may include false positives.
  • 0.50+: Use only when you need very high certainty. May miss many valid targets.
The current value is displayed in blue next to "Current:".
Slice Size (px)
The input resolution passed to the YOLO model. The default is 640, which is the standard training resolution for most YOLOv8 models.
Options: 240, 320, 480, 640, 800, 1280. If you change this, ensure the value is compatible with your model's training configuration.

3. Monitoring Panel - Detailed Explanation

Status
The large text at the top of the Monitoring panel. Possible states:
  • System Ready - The model is loaded and the app is idle, waiting for your command.
  • Processing... - A detection job is currently running.
  • Complete! - The last job finished successfully.
Progress Bar
A horizontal bar that fills from left to right as images are processed. The percentage is shown on the right side of the bar.
Statistics
  • Images: Total number of images in the current batch.
  • Objects: Cumulative count of all woodpiles detected across all processed images so far.
  • Avg Conf: Average confidence score of all detected objects in the current batch.
  • Rate: Average number of objects detected per image.
Processing Log
A scrollable text area with a black background and green text. It displays a timestamped record of every action:
  • Model loading status
  • Folder selections
  • Each image being processed (e.g., [1/3] Processing: test.png)
  • Detection count per image (e.g., Detected 19 objects)
  • Errors or warnings
If something goes wrong, copy the log text here for troubleshooting.

4. Standard Operating Procedure

Step 1 - Wait for Model Load
Launch the application. Within 5-10 seconds, the Model Status in the Configuration panel should change from "Loading..." to "Model Ready ✅" and the Monitoring Status should show "System Ready". Do not proceed until this is complete.
Step 2 - Select Input Folder
Click Browse next to "Input Images Folder". Navigate to the folder containing your source images and click "Open". The log will confirm the selection.
Step 3 - Select Output Folder
Click Browse next to "Output Results Folder". Navigate to (or create) an empty folder for results and click "Open". The log will confirm the selection.
Step 4 - Adjust Parameters
Drag the Confidence Threshold slider to your desired value. For first-time use, leave it at 0.25. If you find too many objects are being missed, lower it to 0.15. If you see too many false detections, raise it to 0.35.
Step 5 - Start Detection
Click the START DETECTION button at the bottom. The Status will change to "Processing...", the progress bar will begin to fill, and the log will show each image being analyzed. Do not close the application during this process.
Step 6 - Review Results
When the "Detection Complete!" dialog appears, click OK. Then click OPEN OUTPUT to open the results folder in Finder. Each result image will have green bounding boxes with labels like Wood 0.85 indicating the object type and confidence score.

5. Troubleshooting

Problem: "Model Ready" never appears
Check that assets/best.pt exists in the application folder. If you moved the app, ensure the model file path is still valid. Try clicking Import and manually selecting the model file.
Problem: 0 objects detected on all images
1. Lower the Confidence Threshold to 0.05 and try again.
2. Verify that your input images actually contain woodpiles.
3. Try re-importing the model file to ensure it is not corrupted.
4. Check the Processing Log for error messages.
Problem: "No images found" error
Ensure your input folder contains .png, .jpg, or .jpeg files. Files starting with detected_ are automatically skipped.
Tip: Batch Processing
You can place dozens of images in the Input Folder and process them all in one run. The app processes them sequentially and saves each result automatically.

6. Keyboard Shortcuts

  • There are no keyboard shortcuts. All actions are performed via mouse clicks.
